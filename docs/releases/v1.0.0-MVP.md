## Problemas Encontrados e Solucoes

### Flyway Validate

O ambiente de desenvolvimento precisou ser ajustado para usar Flyway como responsavel pela criacao do schema e Hibernate em modo `validate`, aproximando dev dos ambientes persistentes.

### @Lob x TEXT

O campo `Registro.relato` apresentava incompatibilidade entre o tipo esperado pelo Hibernate e o tipo `TEXT` criado por migration. A solucao foi manter `columnDefinition = "text"` e remover o uso de `@Lob`.

### Bootstrap Admin

O bootstrap administrativo foi mantido como mecanismo explicito de ambiente, evitando seeds permanentes, credenciais fixas e inserts manuais em migration.

### Swagger Security

O Swagger foi liberado anonimamente apenas nas rotas de documentacao e configurado com os esquemas JWT e API Key, sem tornar APIs de negocio publicas.

### Proxy Nginx

O proxy do frontend foi ajustado para encaminhar corretamente chamadas de API e acessos de arquivos servidos pelo backend.

### CORS

O CORS foi mantido configuravel por variavel de ambiente, evitando dominios fixos no codigo.

### Upload de Video

A validacao de anexos foi consolidada no modulo `registros`, permitindo videos corporativos comuns dentro dos limites definidos e mantendo o storage generico.

### Visualizacao de Anexos

O download de anexos passou a ocorrer por endpoint interno autenticado, validando permissao, vinculo entre registro e anexo e ocultando dados internos como caminho, hash e nome fisico.
