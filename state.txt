# Base+ State

Atualizado em: 2026-06-26

## Estado atual

- Baseline Oficial: `v1.1.0`.
- Plataforma pronta para servir como fundacao de novos projetos de negocio.
- Base+ permanece como fundacao administrativa reutilizavel, sem modulo real de negocio acoplado.
- Backend Spring Boot funcional.
- Frontend React/Vite funcional.
- Auth JWT, refresh token e logout implementados.
- Troca de senha inicial implementada.
- Conta do usuario, preferencias, avatar e sessoes implementados.
- CRUD de usuarios implementado.
- CRUD de roles/perfis implementado.
- CRUD de permissions/permissoes implementado.
- Branding implementado com configuracoes e upload de assets.
- Auditoria implementada com consulta de logs.
- Health e readiness implementados.
- Avatar do menu de conta refinado com moldura visual, halo e indicador online.
- Padrao de novos modulos definido com CRUD Compacto e CRUD Completo em `MODULE_TEMPLATE.md`.
- Padrao arquitetural de integracoes externas definido em `docs/integrations.md`, sem endpoints ou dependencias adicionais.
- Kit visual padrao da marca Base+ organizado em `brand/`, `baseplus-frontend/public/brand/` e `BRAND_GUIDE.md`.
- Kit operacional repetivel criado com `docs/`, `TASK_PROMPT.md` e `scripts/check-project.ps1`.
- Autorizacao por permission definida como granular obrigatoria; roles agrupam permissions, mas nao substituem permissions exigidas.
- Modelo de dois perfis implementado:
  - Perfil funcional agrupa permissions.
  - Perfil organizacional agrupa escopos parametrizaveis.
  - Tipos/unidades organizacionais sao dados configuraveis, nao fixos por codigo.
- CRUD compacto de Estrutura Organizacional implementado em `/app/organizacao`.
- Sequencia de migrations Flyway corrigida, sem duplicidade de versao `V9`.
- Seguranca v1.0.2 aplicada:
  - `JWT_SECRET` obrigatorio fora do perfil `dev`.
  - Uploads aceitam somente `PNG`, `JPG`/`JPEG` e `ICO` validos.
  - Access tokens revalidam usuario ativo/desbloqueado e roles/permissions atuais.
  - Login bloqueia usuario apos 5 tentativas consecutivas com senha invalida.
- Baseline Oficial v1.1.0 consolidada:
  - PostgreSQL consolidado como banco padrao para ambientes persistentes.
  - H2 mantido apenas para desenvolvimento local.
  - Profiles `docker` e `prod` revisados.
  - Flyway ativo e Hibernate validate nos profiles `docker` e `prod`.
  - Teste de compatibilidade PostgreSQL 16 com Testcontainers criado.
  - Dockerizacao completa de backend e frontend criada.
  - Docker Compose integrado criado com PostgreSQL, backend e frontend.
  - PostgreSQL sem porta publica no Compose integrado.
  - Volumes persistentes para PostgreSQL e uploads.
  - Uploads configuraveis por `UPLOAD_DIR`.
  - Readiness `/health/ready` com verificacao real de banco.
  - CORS configuravel por `BASEPLUS_CORS_ALLOWED_ORIGINS`.
  - Vite configuravel por `VITE_ALLOWED_HOSTS` para tuneis locais opcionais.
  - Bootstrap administrativo manual criado para primeira inicializacao de ambientes Docker/HOM/PRD.
  - Documentacao Docker criada em `docs/docker.md`.
  - Checklist oficial v1.1.0 criado em `docs/release-1.1-checklist.md`.
  - Documentacao oficial alinhada ao estado atual da Baseline Oficial v1.1.0.

## Atencoes

- `Promptbaseplus.txt` foi preservado como documento legado de contexto original.
- O repositorio Git possui release inicial publica `v1.0.0`; a correcao Flyway foi publicada em `v1.0.1`; as correcoes de seguranca foram publicadas como `v1.0.2`; a Baseline Oficial da plataforma foi consolidada em `v1.1.0`.
- O segredo JWT padrao permanece apenas no perfil `dev`; esse perfil contem credenciais locais conhecidas e nao deve ser publicado ou exposto externamente.
- Tuneis locais devem ser configurados apenas por `.env` local ou variaveis de ambiente; nao fixar dominios temporarios no codigo.
- Evitar versionar logs, `target`, `dist`, `node_modules`, dumps, backups e uploads locais.
